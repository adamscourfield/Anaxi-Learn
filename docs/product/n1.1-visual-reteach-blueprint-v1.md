# N1.1 Visual Reteach Blueprint (v1)

## Learning target
Recognise place value and digit value reliably up to millions.

## Sequence overview (I do -> We do -> You do)

### Step 1 — Goal card
- Prompt: “Today: find what a digit is worth, not just what it is.”
- Visual: none
- Action: continue

### Step 2 — Visual demo (I do)
- Visual type: `place_value_grid`
- Number: 3,742,915
- Highlight digit 7 column
- Prompt: “The 7 is in hundred-thousands, so value is 700,000.”

### Step 3 — Guided action (We do)
- Visual type: `place_value_grid`
- Number: 5,408,230
- Action: tap the column for digit 4
- Correct: ten-thousands
- Feedback: “Good — now convert place to value: 40,000.”

### Step 4 — Guided action (We do)
- Visual type: `decompose_number`
- Number: 8,030,406
- Action: complete decomposition blanks
- Expected: 8,000,000 + 30,000 + 400 + 6

### Step 5 — Checkpoint 1
- Visual type: `compare_columns`
- Prompt: “In 2,701,450, value of 7 is?”
- Expected: 700,000
- Retry path: hint1 -> hint2 -> alternative explanation

### Step 6 — Near transfer
- Visual type: `compare_columns`
- Prompt: “Which number has 5 worth 50,000?”
- options: [5,203,410 | 1,052,340 | 305,214 | 250,431]
- Expected: 1,052,340

### Step 7 — You do (independent)
- Visual type: minimal support
- Prompt: “Value of 8 in 1,863,205?”
- Expected: 800,000

### Step 8 — Exit transfer check
- Visual type: none
- Prompt: “Write 6,040,070 in expanded form.”
- Expected: 6,000,000 + 40,000 + 70

## Success criteria
- <=1 hint used across steps 5-8
- transfer check correct
- total retries <=3

## Failure path
- If transfer check fails:
  - assign Route C refresher micro-loop (2 steps)
  - then one new transfer item
