# N1.1 Go-Live Readiness Checklist (v1)

Use this to decide **Ready / Not Ready** for N1.1 pilot release.

## A) Content readiness
- [x] N1.1 diagnostic set finalized (4 items) — see `docs/product/n1.1-item-spec-v2.md`
- [x] N1.1 shadow set finalized (6 items, 2 per route) — see `docs/product/n1.1-item-spec-v2.md`
- [x] Every item has metadata tags (`question_role`, `misconception_tag`, `route`, `transfer_level`, `strictness_level`) — aligned in `docs/product/n1.1-item-spec-v2.md` and `docs/product/seeds/n1.1-seed.sql`
- [x] At least 1 non-MCQ item is included in mastery gate cycle — explicit per-route coverage in `docs/product/n1.1-item-spec-v2.md`

## B) Routing readiness
- [x] Route decision service active (`secure` or `A/B/C`) — secure fast-pass now emits via `explanation_route_assigned` payload
- [ ] Route fallback chain works (`A -> B -> C -> intervention`)
- [ ] Transfer signal included in secure decision
- [ ] Route recommendation persisted in diagnostic payload

## C) Input/UX readiness
- [ ] MCQ rendering stable
- [ ] Short text/short numeric rendering stable
- [ ] Input normalization works for entry grading
- [ ] Malformed/missing options fail gracefully (no crash)
- [ ] Student always sees clear next step

## D) Mastery + intervention readiness
- [ ] Shadow pair pass/fail events emitted
- [ ] Not Yet / Developing / Secure transitions are accurate
- [ ] Route C repeated failure triggers intervention flag
- [ ] No mismatch between UI summary and mastery records

## E) Teacher visibility readiness
- [ ] Admin interventions page shows route summary (A/B/C)
- [ ] Shadow pass/fail counts visible
- [ ] Secure fast-pass count visible
- [ ] Intervention flagged count visible

## F) QA readiness
- [ ] Route unit tests pass (N1 routing cases)
- [ ] UI smoke checklist passes (`docs/qa/ui-smoke-checklist-v1.md`)
- [ ] Persona harness passes (secure / misconception / repeated-fail)
- [ ] No stop-ship defects open

## G) Runtime readiness
- [ ] Dependency install and runtime stable on target environment
- [ ] Seed SQL applied (`docs/product/seeds/n1.1-seed.sql`)
- [ ] DB sanity checks pass (skill + 10 linked items)
- [ ] Telemetry events observable in logs

---

## Go / No-Go rule
**GO** when all sections A–G are complete (or one explicitly accepted exception with owner + date).

**NO-GO** if any of these are false:
1. Routing correctness not verified
2. Shadow gate transitions not verified
3. Teacher intervention visibility missing
4. Runtime instability prevents pilot monitoring
