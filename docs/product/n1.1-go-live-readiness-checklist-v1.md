## Go / No-Go rule
**GO** when all sections A–G are complete (or one explicitly accepted exception with owner + date).

**NO-GO** if any of these are false:
1. Routing correctness not verified
2. Shadow gate transitions not verified
3. Teacher intervention visibility missing
4. Runtime instability prevents pilot monitoring
