# N1.1 Production Implementation Tickets (v1)

## Objective
Ship production-ready N1.1 routing layer + question-type handling for UX validation.

## Sprint structure
- **Sprint 1:** Routing + data tags + short entry hardening
- **Sprint 2:** Telemetry + teacher visibility + pilot controls

---

## Backend Tickets

### BE-01: Add item metadata contract
**Task**
- Standardize `Item.options.meta` fields:
  - `question_role`
  - `misconception_tag`
  - `route`
  - `transfer_level`
  - `strictness_level`

**Acceptance criteria**
- Validation utility returns typed metadata or safe fallback.
- Missing metadata does not crash runtime.

---

### BE-02: Implement N1.1 route decision service
**Task**
- Integrate `decideRoute` and fallback chain (`A -> B -> C -> intervention`).
- Trigger after diagnostic bundle scoring.

**Acceptance criteria**
- Deterministic route choice for fixed input.
- Unit tests cover secure/C/B/A fallback cases.

---

### BE-03: Shadow gate evaluator
**Task**
- Evaluate route-linked shadow pair pass/fail.
- Apply mastery transitions:
  - Not Yet / Developing / Secure

**Acceptance criteria**
- 2/2 route shadow pass updates status.
- Secure requires transfer condition.

---

### BE-04: Entry grading normalization
**Task**
- Add normalization pipeline for short entry:
  - trim spaces
  - remove commas
  - case-insensitive aliases

**Acceptance criteria**
- Equivalent valid answers grade correctly.
- Strict mode still available via metadata.

---

### BE-05: Event telemetry extension
**Task**
- Emit route + shadow + mastery events:
  - `route_selected`
  - `shadow_pair_passed/failed`
  - `skill_status_changed`

**Acceptance criteria**
- Event payload always includes `userId`, `skillId`, `route`, `reason`.
- Queryable for pilot dashboard.

---

## Frontend Tickets

### FE-01: Question renderer by `answer_type`
**Task**
- Render component based on answer type:
  - `MCQ`
  - `SHORT_NUMERIC`
  - `SHORT_TEXT`

**Acceptance criteria**
- Each type has validation + submit behavior.
- Unsupported type shows graceful fallback state.

---

### FE-02: Short entry UX hardening
**Task**
- Add inline formatting guidance.
- Show clear input errors without resetting progress.

**Acceptance criteria**
- User can correct and resubmit without page refresh.
- No silent failure states.

---

### FE-03: Route feedback panel
**Task**
- Show student where they are in route flow:
  - “Route A/B/C in progress”
  - “Shadow check 1 of 2”

**Acceptance criteria**
- Students always know next action.
- No extra cognitive load in UI.

---

### FE-04: Session summary enhancements
**Task**
- Add status transition summary:
  - Not Yet -> Developing/Secure
- Include retry/recovery highlights.

**Acceptance criteria**
- Summary reflects backend state exactly.
- No mismatch with mastery record.

---
