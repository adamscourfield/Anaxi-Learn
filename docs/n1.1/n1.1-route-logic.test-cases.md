# N1.1 Route Logic Test Cases

## Required cases

1. Secure fast-pass
- Input: no misconception tags + strong shadow pass
- Expected route: A (default)

2. C-route scenario
- Input misconception tags: m1 or m2
- Expected route: C

3. B-route scenario
- Input misconception tags: m3 or m4
- Expected route: B

4. Mixed fallback to A
- Input misconception tags: m1 + m3 (or unclear)
- Expected route: A

5. Fallback chain
- If shadow fails on A -> next B
- If shadow fails on B -> next C
- If shadow fails on C -> intervention flag
